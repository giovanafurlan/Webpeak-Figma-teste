<p><strong>PÁGINAS AMP</strong> carregam mais rapidamente e possuem mecanismos para garantir a monetização adequada de páginas web para dispositivos móveis. É uma ferramenta de interesse para todos empreendedores digitais que buscam melhorar seus ganhos e ampliar o tráfego orgânico e o SEO de suas páginas mobile. Vamos entender em detalhes o que é como funciona essa ferramenta.</p><p><strong>A velocidade de carregamento de uma página web influencia diretamente na experiência de usuário.</strong></p><p>Páginas que não possuem performance adequada e são pouco responsivas tendem a <strong>A FAZER COM QUE A TAXA DE DESISTÊNCIA DOS USUÁRIOS AUMENTE </strong>(usuário entra no site e sai sem ter realmente navegado pelo conteúdo).</p><p>É nesse contexto que surgiu o AMP. Trata-se de uma plataforma aberta desenvolvida pelo Google para melhorar a performance e velocidade de carregamento de páginas web em dispositivos mobile.</p><blockquote>Leia também: <a href="https://www.webpeak.com.br/blogs/como-maximizar-suas-vendas-com-tecnicas-de-seo" target="_blank">Como maximizar suas vendas com técnicas de SEO</a></blockquote><h2>O que é AMP?</h2><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b5f681b7f6773cb220a7008_o-que-sao-e-como-funcionam-paginas-amp.jpg" alt="páginas AMP" width="auto" height="auto" loading="auto"></div></figure><p><strong>AMP</strong> é a abreviação para <em>Accelerated Mobile Pages</em> (Aceleração de páginas mobile). Em termos gerais, consiste em um framework acessível e amigável &nbsp;para <strong>CRIAÇÃO DE PÁGINAS WEB DE CARREGAMENTO RÁPIDO PARA DISPOSITIVOS MOBILE</strong>.</p><p>É uma IDE open-source que tem o Google como um dos principais patrocinadores. A plataforma foi desenhada para que os usuários e donos de páginas na internet possam facilmente melhorar a velocidade de carregamento e a experiência de usuário em suas páginas.</p><p>Tudo isso sem sacrificar as estratégias de anúncio e cliques promocionais.</p><p>Esse tipo de otimização pode ser obtida por usuários e programadores avançados por meio de constantes melhorias de performance na SEO onpage e no HTML das páginas. Mas a grande maioria dos donos de sites não possuem recursos ou tempo para investir na melhoria de performance contínua dos sites.</p><p>O AMP do Google permite que essas otimizações sejam feitas de forma fácil e sem alterar a experiência de usuário original da página.</p><blockquote>Saiba mais: <a href="https://www.webpeak.com.br/blogs/amp-acelerate-mobile-pages-e-pagespeed-insights-do-google-entenda-a-diferenca" target="_blank">AMP Acelerate Mobile Pages e Pagespeed Insights do Google: Entenda a diferença</a></blockquote><h2>Como AMP funciona?</h2><p>O funcionamento do AMP é baseado em três componentes principais:</p><p><strong>AMP JS: </strong>É a framework Javascript para páginas mobile. É responsável por gerenciar recursos e o carregamento assíncrono(uma velha conhecida técnica para acelerar o carregamento de páginas web). Plataformas <strong>JAVASCRIPT DIFERENTES DA AMP JS NÃO SÃO PERMITIDAS NA AMP</strong>;</p><p><strong>AMP HTML</strong>: É um variante do HTML tradicional. Apesar de algumas tags e propriedades próprias, para quem já está familiarizado com o HTML tradicional não é difícil migrar para o AMP HTML. Detalhes sobre os aspectos específicos da linguagem podem ser conferidos na <a href="https://www.ampproject.org/learn/overview/" target="_blank">página oficial do projeto AMP</a>;</p><p><strong>AMP CDN</strong>: Este é um recurso opcional para melhoria de performance. Após a página estar de acordo com o AMP, o AMP CDN automaticamente faz otimizações e melhorias de performance. É um Content Delivery Network, também chamado de <strong>GOOGLE AMP CACHE. &nbsp;</strong></p><p>Por meio desses três componentes o AMP melhora a performance da página focando nos seguintes aspectos fundamentais:</p><p><strong>CARREGAMENTO ASSÍNCRONO</strong>: A interatividade dos anúncios e a própria renderização do JavaScript pode muitas vezes atrasar o carregamento de página. O AMP JavaScript faz o carregamento assíncrono de forma a não atrasar a renderização da página, carregando conteúdos mais importantes primeiro;</p><p><strong>AJUSTE DE TAMANHOS ESTÁTICO:</strong> Vários recursos externos como imagens, anúncios e iFrames informam ao AMP HTML o seu tamanho e posição antes mesmo de serem carregados. Isso significa que o layout da página é carregado sem ter que esperar pelo download dessas imagens e outros recursos externos;</p><p><strong>EXTENSÕES SÃO IMPEDIDAS DE BLOQUEAR A RENDERIZAÇÃO DO SITE: </strong>OAMP não permite que mecanismos de extensão bloqueiem a renderização do site. Qualquer extensão que requeira requisições HTTP são gerenciadas para não deixar esses requests bloquear a renderização da página;</p><p><strong>RESTRINGE O FUNCIONAMENTO DE JAVASCRIPTS DE TERCEIROS: </strong>Recursos de JavaScript externos geralmente usam carregamento síncrono. O AMP JS restringe a atuação desses JS externos a caixas iFrames, impedindo que eles bloqueiem o funcionamento principal da página;</p><p><strong>TODO CSS É INLINE</strong>: &nbsp;O AMP restringe o tamanho da página CSS à 50 Kilobytes, além de obrigar que somente <strong>CSS DE ESTILO INLINE</strong> seja utilizado. &nbsp;O CSS escrito em inline style está relacionado à utilização da tag <em>style </em>para aplicar um estilo único a apenas um elemento;</p><p><strong>MINIMIZA OS RECÁLCULOS DE ESTILO</strong>: Nas páginas AMP, todas as leituras DOM são feitas antes que todos os procedimentos de escrita. Isto garanti que no máximo um processo de recalculo de estilo página será feito por frame.</p><p><strong>PERMITE APENAS ANIMAÇÕES ACELERADAS POR CPU: </strong>O AMP permite apenas animações que possam ser aceleradas por CPU;</p><p><strong>PRIORIZA O CARREGAMENTO DE RECURSOS:</strong> Por fim, um último elemento importantíssimo é que o AMP gerencia todos os downloads de recursos, dando prioridade maior para aqueles que são mais importantes. Recursos secundários e menos importantes para o usuário são carregados por último.</p><p>Todas essas ferramentas em conjunto resultam em um carregamento de página muito mais rápido, contribuindo para a usabilidade e experiência de usuário.</p><blockquote>Leia mais: <a href="https://www.webpeak.com.br/blogs/tempo-de-carregamento-do-site-dicas-de-como-melhorar-seu-site" target="_blank">Tempo de Carregamento do Site - Dicas de como melhorar seu site</a></blockquote><h2>Como monetizar e medir a performance?</h2><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b5f68707f677369970a700a_o-que-sao-e-como-funcionam-paginas-amp.jpeg" alt="páginas AMP" width="auto" height="auto" loading="auto"></div></figure><p>A melhor experiência de usuário e posicionamento do google são fatores que websites usam para aumentar o <strong>tráfego orgânico</strong> em seu site.</p><p>A maneira clássica de monetizar um site é por meio de anúncios publicitários, sendo que quanto maior o tráfego orgânico(gerado por buscas em buscadores na internet) e a taxa de conversão(número de usuários que realizam um clique em um anúncio ou outra ação desejada) maior é &nbsp;resultado financeiro da página.</p><p>No entanto, uma grande dificuldade surgiu para monetizar essas páginas. Cada vez mais bloqueadores de anúncios tem dificultado a monetização das páginas. Além de bloquear os anúncios, impedindo cliques que renderiam retorno financeiro, o <em>adblockers</em> também acabam funcionando como um acelerador.</p><p>Ao bloquear os vários anúncios e scripts publicitários, o carregamento de página fica mais ráṕido, incentivando ainda mais a utilização dessas extensões.</p><blockquote>Separamos mais um conteúdo para você: <a href="https://www.webpeak.com.br/blogs/webmaster-como-escolher-uma-ferramenta-para-analise-de-performance" target="_blank">Webmaster: Como escolher uma ferramenta para análise de performance</a></blockquote><h3>Como o AMP ajuda a monetizar o seu site?</h3><p>Um dos objetivos que o Google tinha em mente ao <strong>criar o AMP</strong> era exatamente encontrar uma solução para garantir a monetização de páginas web ao mesmo tempo em que garante uma boa experiência de usuário.</p><p>Dessa forma, o AMP oferece suporte para diversos formatos, redes e tecnologias de anúncios específicos para páginas aceleradas. Os principais anunciantes e as principais ferramentas de anúncio já possuem extensões para anúncios compatíveis com o AMP. Dentre eles:</p><p>Amazon A9;Google AdSense;Google Doubleclick;Flite;AdReactor;AOL AdTech;DotAndAds;</p><p>Além dos anúncios tradicionais, o AMP também oferece suporte para implementar paywalls e assinaturas. De maneira geral, o AMP tenta prover um contexto no qual a experiência de usuário seja tão boa quanto se não existissem anúncios ou interrupções publicitárias(oferecendo uma assinatura, por exemplo).</p><blockquote>Leia mais: <a href="https://www.webpeak.com.br/blogs/empresas-estao-enlouquecendo-pelo-amp-para-aceleracao-mobile" target="_blank">Empresas estão enlouquecendo pelo AMP para aceleração mobile</a></blockquote><h3>Como medir a performance?</h3><p>Não é possível administrar algo sem métricas e estatísticas. O mesmo vale para seu site AMP. Como saber se a performance e outros parâmetros estão realmente otimizados?</p><p>O AMP oferece uma ferramenta de Analytics. Para evitar que a mensuração e captação de estatísticas múltiplas torne o site um pouco mais lento, o AMP usa a estratégia de “medir uma vez e reportar muitas vezes”.</p><p>A ferramenta estatística do AMP é baseada em dois componentes:</p><p><strong>AMP-PIXEL ELEMENT</strong>: Trata-se de uma tag usada para contar o número de visitas à página. Funciona como um típico pixel de rastreamento;</p><p><strong>AMP-ANALYTICS EXTENDED COMPONENT</strong>: Este é um <em>feature</em> mais avançado para a configuração de análises e estatísticas. Ao contrário do pixel de rastreamento, permite maior grau de customização e interações com usuários.</p><p>Para usar o Amp-Analytics é necessário adicionar uma biblioteca JavaScript na seção “<em>&lt;head&gt;&lt;/head&gt;</em>” e também configurar o corpo da página com marcações JSON(JavaScript Object Notation).</p><p>Para entender em detalhes o funcionamento do Analytics no AMP, visite <a href="https://www.ampproject.org/docs/reference/components/amp-pixel" target="_blank">essa</a> página. O site oficial do projeto AMP conta com bons exemplos e códigos intuitivos para quem está começando.</p><blockquote>Leia mais:&nbsp;<a href="https://www.webpeak.com.br/blogs/links-patrocinados-e-busca-organica-qual-a-diferenca" target="_blank">Links Patrocinados e Busca Orgânica: Qual a diferença?</a></blockquote><h2>Considerações finais</h2><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b5f68f5746d20ae9343d3db_o-que-sao-e-como-funcionam-paginas-amp%20(2).jpeg" alt="páginas AMP" width="auto" height="auto" loading="auto"></div></figure><p>Para começar a implementar uma página AMP, você tem que manter duas versões de cada conteúdo da página. Uma versão é a página original, como tradicionalmente vista pelos usuários. A outra é a página acelerada com AMP. Uma tag especial é utilizada para identificar a versão original em contraposição com a versão AMP.</p><p>Um ponto importante antes de você começar a implementar é saber que algumas funcionalidades de sua página podem não estar disponíveis na <strong>versão AMP</strong>. Por exemplo, formulários para criação de leads e comentários on-page não são permitidos pelo AMP.</p><p>Isso acontece devido à não utilização de elementos de formulário e JavaScripts diferentes do AMP JS. Apesar desse ponto negativo, já existem algumas soluções para isso utilizando iFrames.</p><p>Devido às <strong>restrições do AMP</strong>, é bem provável que você terá que reescrever o template do site. As limitações de tamanho e a obrigação de todas as inserções CSS serem inline são bons exemplos de aspectos do seu template que terão que ser reescritos.</p><blockquote>Leia também: <a href="https://www.webpeak.com.br/blogs/a-importancia-do-web-mobile-para-e-commerce" target="_blank">A importância do Web Mobile para E-commerce</a></blockquote><p>A forma de tratar elementos multimídia como imagens também é diferente. Deve-se especificar explicitamente a altura e comprimento utilizando as tags e elementos do AMP, como o “<em>&lt;amp-img&gt;&lt;/amp-img&gt;</em>” e o “<em>&lt;amp-anim&gt;&lt;/amp-anim&gt;</em>”(para animações GIFS). Essas são as tags para inserção de imagens e animações, respectivamente.</p><p>Apesar de demandar um estudo inicial e algumas tentativas até que tudo fique dentro das especificações corretas, ter uma <strong>página AMP</strong> garante uma melhor experiência de usuário e um carregamento ágil e sem bugs. Fatores fundamentais para qualquer estratégia de empreendedorismo digital ou mesmo páginas institucionais.</p><p>Com este artigo você já sabe o que é e como funciona.
