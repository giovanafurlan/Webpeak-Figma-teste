<p>O Google tem investido de forma consistente em seu projeto AMP - Accelerated Mobile Pages. A ferramenta é vista como uma reação aos recentes movimentos de Facebook (Facebook Instant Articles) e Apple(Apple News).Introdução</p><p>O Google tem investido de forma consistente em seu projeto <a href="https://www.webpeak.com.br/blogs/4-acoes-de-mobile-marketing-para-voce-destacar-sua-empresa" target="_blank"><strong>AMP - ACCELERATED MOBILE PAGES</strong></a>. A ferramenta é vista como uma reação aos recentes movimentos de Facebook (<em>Facebook Instant Articles</em>) e Apple(<em>Apple News</em>).</p><p>As principais empresas de tecnologia vem tentando conquistar um quinhão cada vez maior dos investimentos publicitários na web e ferramentas visando a <a href="https://www.webpeak.com.br/blogs/quanto-custa-anunciar-no-google-tire-todas-suas-duvidas" target="_blank"><strong>construção de páginas otimizadas para anúncios</strong></a> e com alta performance vem sendo lançadas e aprimoradas.</p><p>O AMP é um Framework de código aberto(<em>open-source</em>) cujo objetivo é <strong>MELHORAR O DESEMPENHO NA NAVEGAÇÃO NA INTERNET MÓVEL</strong>. A intenção é que as páginas web voltadas para o acesso por dispositivos móveis sejam capazes de abrigar conteúdos variados e de qualidade(vídeos, animações, gráficos, imagens) ao mesmo tempo que possam receber anúncios inteligentes. Tudo isso com um tempo extremamente exíguo de carregamento.</p><blockquote>Leia também: <a href="https://www.webpeak.com.br/blogs/quem-deve-se-preocupar-em-ter-paginas-amp-2">Quem deve se preocupar em ter páginas AMP?</a></blockquote><h2>Utilizando o AMP HTML</h2><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b4cd803fb7617079d1cbd8d_aprenda-a-criar-a-sua-pagina-amp-html.jpeg" alt="página AMP HTML"></div></figure><p>Ao configurar uma <strong>página AMP</strong> basicamente você está criando uma segunda versão com o mesmo conteúdo da original, só que otimizada para ser carregada em dispositivos mobile. Essa versão AMP possui a identificação “<em>/amp</em>” ao final do endereço URL.</p><p>Como duas versões da páginas estarão em funcionamento, é necessário utilizar um tag especial para avisar ao google que não se trata de conteúdo duplicado, mas sim de versões com e sem AMP. &nbsp;<strong>A PÁGINA COM O CONTEÚDO ORIGINAL DEVE CONTER UMA CANONICAL TAG</strong>. Essa tag é colocada no início da página e consiste na seguinte linha:</p><p><strong>&lt;link rel="”CANONICAL”" href="”WWW.VERSAOORIGINAL.COM.BR”"&gt;</strong></p><p>Basicamente uma referência apontando diretamente para a versão com conteúdo original na qual a página AMP é baseada.</p><p>O AMP é composto de três partes principais, sendo uma delas o AMP HTML. Uma variante do HTML tradicional focada em otimizar o desempenho da página. Vamos ver um pouco melhor como trabalhar o AMP HTML para acelerar a sua página web. Vamos lá?</p><h3>Quais sites podem utilizar AMP?</h3><p>O projeto AMP está aberto para qualquer tipo de site. A princípio, foram grandes veículos de comunicação como grandes grupos e portais de tecnologia(GIZMODO, eBay) os primeiros a utilizar. Com o tempo, cada vez mais páginas vem sendo otimizadas com AMP.</p><p>Os cada vez mais populares E-commerces também são um público alvo importante. Um carregamento lento pode fazer o usuário desistir de visitar uma loja online. Além de poder ser aplicado a qualquer tipo de site e portal, o projeto é <em>open-source</em>, e dependendo das habilidades de sua empresa, é possível implementar versões customizadas.</p><p>É possível aplicar o AMP a páginas específicas também, como catálogos de produtos, homes e outras páginas que sejam mais importantes.</p><h3>Fazendo AMP em HTML</h3><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b4cd8891c15d69f0865d7f0_aprenda-a-criar-a-sua-pagina-amp-html%20(2).jpeg" alt="página AMP HTML"></div></figure><p>Para aprendermos os principais pontos do AMP HTML vamos utilizar o template exemplo que o próprio projeto oficial utiliza,com algumas modificações para facilitar o entendimento e leitura.</p><p>Vamos explicar linha por linha como cada elemento de uma página AMP HTML funciona. O exemplo abaixo é um código típico de uma página AMP. Todas as features encontradas aqui são as que você deve configurar na sua versão AMP.</p><p>&lt;!doctype html&gt;&lt;<strong>HTML</strong> amp lang="en"&gt; &nbsp;&lt;<strong>HEAD</strong>&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>META</strong> charset="utf-8"&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>SCRIPT</strong> async src="<strong>https://cdn.ampproject.org/v0.js</strong>"&gt;&lt;/<strong>SCRIPT</strong>&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>TITLE</strong>&gt;Hello, AMPs&lt;/<strong>TITLE</strong>&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>LINK</strong> rel="canonical" href="<strong>http://versaooriginal.com.br</strong>" /&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>META</strong> name="viewport" content="width=device-width,minimum-scale=1,initial-scale=1"&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>SCRIPT</strong> type="application/ld+json"&gt; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;{&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; ...script &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;} &nbsp;&nbsp;&nbsp;&lt;/<strong>SCRIPT</strong>&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>STYLE</strong> amp-boilerplate&gt;<strong>BODY</strong>{...o código AMP Boilerplate vem aqui... }&lt;/<strong>STYLE</strong>&gt;&lt;/<strong>NOSCRIPT</strong>&gt; &nbsp;&lt;/<strong>HEAD</strong>&gt; &nbsp;&lt;<strong>BODY</strong>&gt; &nbsp;&nbsp;&nbsp;&lt;<strong>H1</strong>&gt;Welcome to the mobile web&lt;/<strong>H1</strong>&gt; &nbsp;&lt;/<strong>BODY</strong>&gt;&lt;/<strong>HTML</strong>&gt;</p><p><strong>&lt;!DOCTYPE HTML&gt; &nbsp;- </strong></p><p>Toda página AMP HTML obrigatoriamente deve começar com essa marcação;<strong>&lt;html amp="" lang="EN"&gt;&lt;/html&gt;</strong> - Também é obrigatório que no topo de uma página AMP contenha essa marcação. Ela identifica que o conteúdo desse site é AMP;<strong>&lt;head&gt;&lt;/head&gt;</strong> e <strong>&lt;body&gt;&lt;/body&gt;</strong> - Essas são tags opcionais no HTML tradicional, mas são obrigatórias no AMP HTML;<strong>&lt;meta charset="UTF-8"&gt;</strong> - Esta tag deve vir logo em seguida à tag &lt;head&gt;. É usada para identificar a codificação da página(ut-8 no caso);&lt;/head&gt;<strong>&lt;script async src="HTTPS://CDN.AMPPROJECT.ORG/V0.JS"&gt;&lt;/script&gt; - </strong></p><p>Marcação para incluir a biblioteca AMP JS (Java Script);<strong>&lt;link rel="CANONICAL" href="HTTP://VERSAOORIGINAL.COM.BR"&gt; - </strong></p><p>marcação que aponta para a versão original do website;<strong>&lt;meta name="VIEWPORT" content="WIDTH=DEVICE-WIDTH,MINIMUM-SCALE=1,INITIAL-SCALE=1"&gt; - </strong></p><p>Tag responsável por identificar um viewport responsivo. Páginas responsivas são páginas que se adequam bem a diferentes tamanhos de tela. Para criar uma página AMP responsiva;<strong><em>CÓDIGO AMP BOILERPLATE</em></strong> - Este é um trecho de código extenso que pode ser visto por completo <a href="https://www.ampproject.org/docs/reference/spec/amp-boilerplate">aqui</a>. &nbsp;Este código é responsável por esconder conteúdos até que o AMP JS tenha sido carregado corretamente.</p><p>Este é o conjunto de tags padrão do AMP HTML. Vamos ver agora como adicionar imagens e configurar o Layout padrão de sua página acelerada.</p><h3>Adicionando imagens no AMP HTML</h3><p>A adição de imagens é um bom exemplo de como o AMP HTML é ligeiramente diferente do HTML tradicional. A tag na linguagem HTML convencional para adicionar imagens é <em>&lt;img&gt;</em>. Já no AM HTML a tag é um pouquinho diferente: “&lt;<em>amp-img&gt;”</em>. &nbsp;Para entender bem a diferença, veja o exemplo abaixo em AMP HTML:</p><p>&lt;<strong>AMP-IMG</strong> src="bemvindo.jpg" alt="bemvindo" height="600" width="400"&gt;&lt;/<strong>AMP-IMG</strong>&gt;</p><p>Este código adiciona uma imagem de nome bemvindo, do tipo .jpg, com altura e comprimento conforme especificados.</p><h3>Configurando Layout de apresentação no AMP HTML</h3><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b4cd8c727690c393c83375b_aprenda-a-criar-a-sua-pagina-amp-html.jpg" alt="página AMP HTML"></div></figure><p>Como toda página web, páginas AMP usam propriedades do CSS para implementar elementos de estilo. A diferença é que páginas AMP devem ter obrigatoriamente apenas elementos CSS <strong>INLINE</strong>. Isto é feito usando a tag “<em>&lt;amp-custom&gt;”&lt;/amp-custom&gt;</em>, dentro da tag “<em>&lt;head&gt;&lt;/head&gt;</em>”.Veja o exemplo abaixo:</p><p><em>&lt;</em><strong><em>STYLE</em></strong><em>amp-custom&gt; &nbsp;/* qualquer estilo customizado deve vir aqui */ &nbsp;</em><strong><em>BODY</em></strong><em> { &nbsp;&nbsp;&nbsp;</em><strong><em>BACKGROUND-COLOR</em></strong><em>: </em><strong><em>BLUE</em></strong><em>; &nbsp;} &nbsp;</em><strong><em>AMP-IMG</em></strong><em> { &nbsp;&nbsp;&nbsp;</em><strong><em>BACKGROUND-COLOR</em></strong><em>: </em><strong><em>GREY</em></strong><em>; &nbsp;&nbsp;&nbsp;</em><strong><em>BORDER</em></strong><em>: 2</em><strong><em>PXSOLIDBLACK</em></strong><em>; &nbsp;}&lt;/</em><strong><em>STYLE</em></strong><em>&gt;</em></p><p>Apenas um stylesheet é permitido por página AMP. E também há algumas restrições em relação aos seletores utilizados. Para entender melhor quais são as restrições em relação aos estilos do CSS, consulte o <a href="https://www.ampproject.org/docs/guides/responsive/style_pages">site oficial do projeto AMP</a>.</p><h3>Visualizando a sua página acelerada</h3><p>Um preview da página acelerada pode ser visto como qualquer outro arquivo .html. Não é necessário compilar ou fazer qualquer tipo de verificação. Caso haja erros, a página simplesmente será apresentada com os erros ou não será visualizada.</p><p>São duas maneiras para visualizar a sua página AMP HTML.</p><p>Abra a página diretamente no seu browser a partir do arquivo em seu sistema. Nesse caso, alguns elementos podem não funcionar devido a possíveis falhas nos <em>XMLHttpResquests</em>; Usar um servidor web local como Apache 2 ou Nginx;</p><p>Visualizar sua página não te isenta da responsabilidade de validar a página porém. Mesmo que a visualização esteja aparentemente ok, é preciso validar o código de forma adequada em busca de erros.</p><h3>Validando o HTML para AMP</h3><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b4cd92c15cec23f1e1300ce_aprenda-a-criar-a-sua-pagina-amp-html%20(3).jpeg" alt="página AMP HTML"></div></figure><p>Após configurar o página web para o AMP HTML, ainda é preciso fazer uma validação para garantir que todo o código está conforme as regras AMP. Aqui temos uma boa notícia. Os desenvolvedores do projeto oficial criaram e disponibilizaram gratuitamente uma ferramenta para validar o AMP HTML.</p><p>O nome da ferramenta é <strong>AMP VALIDATOR</strong> e está disponível <a href="https://validator.ampproject.org/">aqui</a>. Basta copiar e colar o link da página AMP lembrando de acrescentar o “/amp” ao final e dar o comando “validate”, mostrado em destaque no validador.</p><p>Na parte inferior é mostrada uma barra de Status. Se a mesma apresentar a mensagem “PASS”, significa que seu código HTML está validado. Caso a mensagem exibida seja “FAIL”, significa que seu código contém erros e precisa ser verificado novamente. A grande vantagem do AMP Validator é que ele apresenta de forma bem intuitiva uma lista dos erros encontrados nos casos em que a validação não é bem sucedida.</p><p>Uma outra forma de validar o código é pelo console <strong>CHROME DEVTOOLS</strong>. Siga os seguintes passos:</p><p>Abra sua página em seu web browser(Google Chrome);Adicione o trecho “<em>#development=1</em>” à URL da página como mostrado abaixo:<strong><em>http://localhost:8000/released.amp.html#development=1</em></strong>Abra o &nbsp;console Chrome DevTools e verifique por erros de validação.</p><p>A etapa de validação é importantíssima. Se seu código AMP HTML possuir erros, ele simplesmente não será descoberto e distribuído por plataformas de busca como Google.</p><h3>Fazendo acompanhamento de acessos e indexação</h3><p>Por fim, após todo o trabalho para implementar sua página AMP HTML e validá-la, é necessário acompanhar as estatísticas de acesso e indexação. Para isso você vai precisar acessar o <strong>GOOGLE SEARCH CONSOLE</strong>(ou seja, você precisa ter uma conta nessa ferramenta).</p><p>No console, todas as informações são apresentadas de forma transparente para o usuário. Basta seguir o menu “<em>Aspecto de pesquisa</em>” e selecionar “<em>Páginas aceleradas para dispositivos móveis</em>”.</p><p>Aqui você terá acesso à evolução do número de páginas indexadas ao Google e também qualquer erro que eventualmente elas possam ter(um recurso muito útil).</p><blockquote>Leia também: <a href="https://www.webpeak.com.br/blogs/como-o-seo-vai-trazer-seu-primeiro-pedido">Como o SEO vai trazer seu primeiro pedido?</a> </blockquote><h3>Conclusão</h3><figure class="w-richtext-figure-type-image w-richtext-align-floatleft" style="max-width:30%" data-rt-type="image" data-rt-align="floatleft" data-rt-max-width="30%"><div><img src="https://uploads-ssl.webflow.com/5a4661af7b791a00019cfaee/5b4cd95faecadeca46764637_aprenda-a-criar-a-sua-pagina-amp-html%20(4).jpeg" alt="criar página AMP HTML"></div></figure><p>O carregamento de página é um fator fundamental para o acesso a conteúdos em dispositivos móveis.
