<p>Olá, tudo certo?</p><p>Se você acompanha nosso blog ou se já pesquisou sobre SEO para ecommerce no Google, com certeza, já se deparou com algum conteúdo nosso. Como nossa motivação sempre foi compartilhar conhecimento, desenvolvemos uma série de conteúdos que irão facilitar e simplificar ainda mais o dia-a-dia do lojista e usuários da plataforma.</p><p>Iremos começar por uma questão que afeta diversas etapas dos projetos, desde o cadastro inicial, migração e renovação de estoque ou coleção, por exemplo.</p><p>Já pensou subir uma carga de produtos para a loja de uma vez? Isso é possivel via planilha na plataforma VTEX.</p><p>Vamos ao tutorial!</p><blockquote><em>Leia também:</em> <a href="https://www.webpeak.com.br/blogs/long-tail-seo-ficou-mais-facil-de-entender" target="_blank" data-rt-link-type="external"><em>Long Tail SEO: Ficou mais fácil de entender</em></a></blockquote><h2>Cadastro de produtos Via Planilha </h2><p>Primeiro acesse o módulo Ecommerce, que é o modulo onde se gerencia o cadastro de produtos, categorias e marcas.</p><figure class="w-richtext-figure-type-image w-richtext-align-normal" data-rt-type="image" data-rt-align="normal"><div><img src="https://daks2k3a4ib2z.cloudfront.net/58743129b8dc90923ca299a9/587d45ad885bbaa52035eae7_acesso-modulo-ecommerce.png" alt="Cadastro de produtos Via Planilha VTEX"></div></figure><p>Em seguida, na aba Cadastro de Produtos, vá na opção Produtos e SKUs em seguida clique em Importar/Exportar Produtos e Skus.</p><figure class="w-richtext-figure-type-image w-richtext-align-normal" data-rt-type="image" data-rt-align="normal"><div><img src="https://daks2k3a4ib2z.cloudfront.net/58743129b8dc90923ca299a9/587d4668f8ac26a30cd45572_importacao-produtos-vtex.png" alt="Cadastro de produtos Via Planilha VTEX"></div></figure><p>Você será direcionado a uma página onde poderá exportar uma planilha para servir como base para a importação dos produtos. Para isso clique no link Exportação/Importação. Depois vá na aba Exportação e adicione seu email para receber a planilha 'modelo'.</p><figure class="w-richtext-figure-type-image w-richtext-align-normal" data-rt-type="image" data-rt-align="normal"><div><img src="https://daks2k3a4ib2z.cloudfront.net/58743129b8dc90923ca299a9/587d47139f9253b40c1732f6_importacao-produtos-vtex-2.png" alt="Cadastro de produtos Via Planilha VTEX"></div></figure><h4><strong>Existem números próximos aos campos com a seguinte legenda que determinam quais campos deverão ser preenchidos na nova carga de produtos:</strong></h4><p><strong>[1]</strong>&nbsp;Campos obrigatórios na importação de novos produtos.</p><p><strong>[2]</strong>&nbsp;Campos que não devem ser alterados em atualização de&nbsp;<em>produtos já existentes</em>, (esses campos só devem ser preenchidos para novos produtos).</p><p>É importante preencher corretamente os campos para que a importação aconteça rapidamente. Veja qual a informação deve ser inserida na coluna.</p><p><strong>SkuId&nbsp;[2]</strong></p><p>Esta coluna indica se a operação realizada é a adição ou alteração de um SKU, caso preenchida, o SKU com o ID indicado será atualizado com dados contidos na planilha. Para um novo SKU, deixe a coluna em branco.</p><p><strong>NomeSku&nbsp;[1]</strong></p><p>Nome que identifique o SKU dentre os demais SKUs pertencentes ao mesmo produto. Caso seja diferente do nome do produto, será exibido no site ao final do nome do produto. Ex: Produto A Sku 1.</p><p><strong>SkuAtivo&nbsp;[2]</strong></p><p>Define se o SKU existente está ativo ou não.</p><p><strong>SkuEan&nbsp;[1]</strong></p><p>Código de identificação única do Sku(código de barras), aceita até 13 caracteres numéricos. Caso o campo “CodigoReferenciaSKU” tenha sido preenchido, este campo deixa de ser obrigatório.</p><p><strong>Altura&nbsp;[1]</strong></p><p>Altura para cálculo do frete, em centímetros. Exemplo: Para 1 metro, inserir 100,00 neste campo.</p><p><strong>AlturaReal</strong></p><p>Altura real do produto, em centímetros. Exemplo: para 1 metro, inserir 100,00 neste campo.</p><p><strong>Largura&nbsp;[1]</strong></p><p>Largura para cálculo do frete, em centímetros. Exemplo: Para 75 centímetros, inserir 75,00 neste campo.</p><p><strong>LarguraReal</strong></p><p>Largura real do produto, em centímetros. Exemplo: Para 80 centímetros, inserir 80,00 neste campo.</p><p><strong>Comprimento&nbsp;[1]</strong></p><p>Comprimento para cálculo do frete, em centímetros. Exemplo: Para 20 metros, inserir 2000,00 neste campo.</p><blockquote><em>Não deixe de ler:</em> <a href="https://www.webpeak.com.br/blogs/link-building" target="_blank" data-rt-link-type="external"><em>Link Building</em></a></blockquote><p><strong>Peso&nbsp;[1]</strong></p><p>Peso para cálculo do frete. Em gramas; exemplo: Para 5kg, inserir 5000,00 neste campo.</p><p><strong>PesoReal</strong></p><p>Peso real do produto, em gramas. Exemplo: Para 5 kg, inserir 5000,00 neste campo.</p><p><strong>UnidadeMedida</strong></p><p>Utilizado apenas em casos onde é necessário converter a unidade de medida para a venda. Exemplo: O produto é vendido em caixas, porém os clientes querem comprar por metro quadrado. Nos casos comuns, utiliza-se “un”.</p><p><strong>MultiplicadorUnidade&nbsp;[1]</strong></p><p>Seguindo o exemplo do tópico anterior: É o fator que “transforma” caixas em metro quadrado. Nos casos comuns, utiliza-se “1,00”.</p><p><strong>CodigoReferenciaSKU&nbsp;[1]</strong></p><p>Código de referência criado para ajudar na organização.Importante: Após ter feito pela primeira vez, não é possível editar via planilha. Cada SKU deve conter um código de referência diferente. Caso o campo “SKUEan” tenha sido preenchido, este campo deixa de ser obrigatório.</p><p><strong>ValorFidelidade</strong></p><p>Crédito que o cliente recebe ao concluir uma compra de 1 unidade de um determinado SKU. Por exemplo, ao colocar “1,00” neste campo, o cliente ganha um crédito de R$1,00 no site.</p><p><strong>DataPrevisaoChegada</strong></p><p>Em caso de pré-venda, inserir a data prevista de chegada. Exemplo: dd/mm/aaaa.</p><p><strong>CodigoFabricante</strong></p><p>Código fornecido pelo fabricante.</p><p><strong>IdProduto&nbsp;[2]</strong></p><p>Esta coluna indica se a operação realizada é a adição ou alteração de um Produto, caso preenchida, o Produto com o ID indicado será atualizado com os dados contidos na planilha. Caso não for preenchida, indica a criação de um novo Produto.</p><p><strong>NomeProduto&nbsp;[1]</strong></p><p>Nome que identifique o Produto dentre os demais produtos.&nbsp;<strong>Lembre-se!</strong>&nbsp;Esse campo tem um limite de 150 caracteres.</p><p><strong>NomeComplemento</strong></p><p>Descrição curta do Produto, pode ser exibida tanto na página do produto quanto na prateleira.</p><p><strong>ProdutoAtivo&nbsp;[2]</strong></p><p>Define se o Produto existente está ativo ou não.</p><p><strong>CodigoReferenciaProduto</strong></p><p>Código de referência criado para ajudar na organização. Cada produto deve conter um código de referência diferente.</p><p><strong>ExibirNoSite</strong></p><p>“SIM” para o produto ser exibido no site, ou “NÃO” caso queira deixar oculto. Este campo serve para controlar a exibição ou não de um produto, ainda que ele esteja ativo.</p><p><strong>TextoLink&nbsp;[2]</strong></p><p>URL do produto.&nbsp;<a href="http://www.lojaexemplo.com.br/exemplo-2/p" rel="nofollow" data-rt-link-type="external"><strong><em>www.lojaexemplo.com.br/exemplo-2/p</em></strong></a>&nbsp;– neste exemplo, devemos inserir “exemplo-2″ neste campo. Dica: sempre que possível, utilize um textolink simples e amigável. Após ter feito pela primeira vez, não é possível editar via planilha. Cada produto deve conter um textolink diferente.</p><p><strong>DescricaoProduto</strong></p><p>É a descrição. Seja criativo e dê as principais informações sobre o produto. Em caso de não preenchimento, o sistema replicará o “NomeProduto” neste campo para completar o cadastro.</p><p><strong>DataLancamentoProduto</strong></p><p>Campo utilizado para agendar a data de ativação de um produto. Exemplo: dd/mm/aaaa.</p><p><strong>PalavrasChave</strong></p><p>Palavra substituta, ou seja, um ou mais termos que fazem referência ao produto (exemplo:&nbsp;<em>Geladeira -&gt; Refrigerador</em>&nbsp;/<em>Nike -&gt; Naique</em>). Este campo serve para otimizar a ferramenta de busca da loja, tornando-a mais assertiva quando o cliente digita algum termo errado ou alternativo.</p><p><strong>TituloSite</strong></p><p>É o nome que aparece como título no Google e também na aba do navegador quando entramos na página do produto. Em caso de não preenchimento, o sistema replicará o “NomeProduto” neste campo para completar o cadastro.</p><p><strong>MetaTagDescription</strong></p><p>É a descrição do produto para o Google. Aconselhável utilizar até 150 caracteres. Em caso de não preenchimento, o sistema replicará o “NomeProduto” neste campo para completar o cadastro.</p><p><strong>IdFornecedor</strong></p><p>Id do fornecedor. Encontrado na URL da página de alteração de fornecedor no Admin.</p><p><strong>ExibeSemEstoque&nbsp;[1]</strong></p><p>“SIM” para aparecer “produto esgotado” ou “NÃO” para o SKU não ser exibido quando acabar o estoque. Caso seja adotado “SIM”, será exibido o botão “AVISE-ME QUANDO CHEGAR”, no lugar do botão “COMPRAR”.</p><p><strong>Kit&nbsp;[2]</strong></p><p>Define se o SKU do Produto em questão é um Kit.</p><p><strong>IdDepartamento&nbsp;[2]</strong></p><p>Id da categoria de maior nível hierárquico do Produto. Encontrado na URL da página de alteração da categoria no Admin.</p><p><strong>NomeDepartamento&nbsp;[2]</strong></p><p>Nome da categoria de maior nível hierárquico do Produto.</p><blockquote><em>Separamos este artigo para você:</em> <a href="https://www.webpeak.com.br/blogs/a-importancia-do-web-mobile-para-e-commerce" target="_blank" data-rt-link-type="external"><em>A importância do Web Mobile para E-commerce</em></a></blockquote><p><strong>IdCategoria&nbsp;[1]</strong></p><p>Id da categoria de menor nível hierárquico do Produto. Encontrado na URL da página de alteração da categoria no Admin.</p><p><strong>NomeCategoria&nbsp;[2]</strong></p><p>Nome da categoria de menor nível hierárquico do Produto.</p><p><strong>IdMarca&nbsp;[1]</strong></p><p>Id da marca. Encontrado na URL da página de alteração da marca no admin.</p><p><strong>Marca&nbsp;[2]</strong></p><p>Nome da Marca</p><p><strong>PesoCubico</strong></p><p>Este campo não é mais utilizado para cálculo na Logística. Deve ser deixado em branco.</p><p><a href="http://help.vtex.com/hc/pt-br/articles/215891957-Como-cadastrar-condi%C3%A7%C3%A3o-comercial" target="_blank" data-rt-link-type="external"><strong>CondicaoComercial</strong></a></p><p>Condição associada ao Sku do Produto em questão, pode ser utilizada para definir promoções/regras de parcelamento específicas para Skus com a mesma condição.</p><p><strong>CodigosLojas</strong></p><p>Código da loja, caso só tenha uma loja administrada no admin, inserir “1” neste campo; caso tenha mais de uma (lojas no formato multidomínio), colocar o número da loja desejada.</p><p><strong>Acessorios&nbsp;[2]</strong></p><p>Contém os IDs de SKUS relacionados ao SKU em questão como Acessórios.</p><p><strong>Similares&nbsp;[2]</strong></p><p>Contém os IDs de SKUs relacionados ao SKU em questão como Similares.</p><p><strong>Sugestoes&nbsp;[2]</strong></p><p>Contém os IDs de SKUs relacionados ao SKU em questão como Sugestões.</p><blockquote>informações disponibilizadas pela plataforma VTEX, no link:&nbsp;<a href="http://help.vtex.com/tutorial/cadastrando-produto/" target="_blank" data-rt-link-type="external">http://help.vtex.com/tutorial/cadastrando-produto/</a></blockquote><p>Depois de alterar o documento, faça o mesmo processo, mas clique na aba Importação, faça o upgrade da planilha e clique no botão Importar.</p><p>Em seguida irá aparecer uma mensagem na tela e será enviada por e-mail do usuário logado.</p><h4><strong>Lembre-se: Dependendo do tamanho do arquivo, a importação poderá levar alguns minutos para concluir o cadastro de produtos na Vtex.</strong></h4><p>Continue nos acompanhando e veja como fazer Importação de produtos nas mais divertsas plataformas de Ecommerce.</p><p>Até a próxima!
